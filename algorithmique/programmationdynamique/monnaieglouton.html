Rendu de monnaie

Algorithmique : programmation dynamique

  • Révisions

  • Approche Dynamique

L’approche gloutonne du problème de rendu de monnaie est rapide mais ne garantit pas, dans tous les cas, de fournir une solution optimale.
La programmation dynamique optimise l’approche exhaustive pour fournir la meilleure solution au problème.

Révision de l’Algorithme Glouton

L’algorithme glouton est une méthode qui fait des choix locaux optimaux pour résoudre un problème. Pour le rendu de monnaie, il choisit toujours la pièce de la plus grande valeur possible. 

                    def rendu_glouton(somme: int, systeme: list) -> list:
    res = []
    i_piece = 0
    while somme > 0:
        # si pièce est trop grande
        if systeme[i_piece] > somme:
            # on avance dans le système
            i_piece = i_piece + 1
        else:
            res.append(systeme[i_piece])
            somme = somme - systeme[i_piece]
    return res
Algorithme Naïf

L’algorithme naïf explore toutes les possibilités pour trouver la solution optimale. 

            def rendu_naif(somme: int, systeme: list) -> list:
  if somme > 0:
      # pire solution: somme = 1 + 1 + 1 ...
      mini = [1 for _ in range(somme)]  
      # Teste tous les possibles pour chaque pièce
      for piece in systeme:
          if piece <= somme:
              # descente dans une branche
              solution = [piece] + \
                      rendu_naif(somme-piece, systeme)
              if len(solution) < len(mini):
                  # une meilleure solution
                  mini = solution
      # garde le rendu minimum
      return mini
  else:
      return []
Approche Top-Down (Descendante)

Utilise la récursion avec mémoïsation pour stocker les solutions déjà calculées. 

              def rendu_td(somme: int, systeme: list, track: list) -> list:
                # le choix de pièces a déjà été calculé
                if len(track[somme]) > 0:
                    return track[somme]
                elif somme > 0:
                    mini = [1 for _ in range(somme)] 
                    # Teste tous les possibles pour chaque pièce
                    for piece in systeme:
                        if piece <= somme:
                            pieces = [piece] + rendu_td(somme-piece, systeme, track)
                            if len(pieces) < len(mini):
                                mini = pieces
                    # garde le rendu minimum
                    track[somme] = mini
                    return mini
                else:
                    return []
Approche Bottom-Up

Utilise une approche itérative pour résoudre d’abord les petits problèmes. 

              def rendu_bu(somme: int, systeme: list) -> list:
    track = [[] for _ in range(somme+1)]
    # pour chaque pièce de track on cherche le nombre minimum de pièces à rendre
    for x in range(1, somme+1):
        # pire solution: ne rend que des pièces de 1
        mini = [1 for _ in range(x)]
        for piece in systeme:
            if piece <= x:
                # on utilise les solutions précédentes
                solution = [piece] + track[x-piece]
                if len(solution) < len(mini):
                    mini = solution
        # mémorise solution
        track[x] = mini

    return track[somme]
Conclusion

La programmation dynamique est une technique efficace pour résoudre des problèmes d’optimisation comme le rendu de monnaie. Elle permet de trouver des solutions optimales en évitant les recalculs inutiles grâce à la mémoïsation. Ce cours montre l’importance de choisir la bonne stratégie algorithmique en fonction du contexte et des contraintes du problème.
